## Project Overview

`create-prompt` is a CLI tool for creating organized AI prompts with smart auto-filling and context detection. It's a zero-dependency Node.js CLI.

## Commands

```bash
# Run CLI locally during development
node bin/create-prompt

# Test specific commands
node bin/create-prompt --help
node bin/create-prompt init
node bin/create-prompt persona            # list personas
node bin/create-prompt persona backend    # install one (--force to overwrite)
node bin/create-prompt project-structure
node bin/create-prompt files-markdown
node bin/create-prompt finish
```

No build step required - pure Node.js with CommonJS modules.

## Architecture

### Layered Structure

```
CLI Layer (src/cli.js)
    ↓ routes commands
Commands (src/commands/*.js)
    ↓ orchestrate
Services (src/services/*.js)
    ↓ use
Utilities (src/utils/*.js)
```

### Key Modules

**CLI Entry**: `bin/create-prompt` → `src/cli.js` → `createCLI().run(args)`

**Commands** (`src/commands/`):
- `createPrompt.js` - Interactive prompt creation
- `initCommand.js` - Project setup (scaffold, project info, instructions, .gitignore, then runs generateStructure)
- `personaCommand.js` - List/install persona templates into `context/persona.md`
- `finishCommand.js` - Git commit workflow
- `generateStructure.js` - Project tree generation + recent activity refresh + base prompt rebuild
- `generateFilesMarkdown.js` - File content aggregation (also exports `ensureRequestedFilesExists`)

**Services** (`src/services/`):
- `contextService.js` - Parses `.create-prompt/context/*.md` files with YAML frontmatter; `rebuildBasePrompt()` assembles `base_prompt.md` (preserves `## History` across rebuilds)
- `templateService.js` - Base prompt template handling; `buildFromBasePrompt()` fills title, date, and `config.TASK_PLACEHOLDER`
- `promptService.js` - Prompt file creation
- `projectService.js` - Project metadata detection (`getProjectInfo`, `buildProjectInfoMarkdown`)
- `gitContextService.js` - Git state reading (`getGitState`, `buildRecentActivityMarkdown` via execSync) + commit relevance filtering
- `relevanceService.js` - Context relevance scoring
- `taskTypeService.js` - Task type detection
- `requirementsService.js` - Task requirements inference
- `fileSuggestionService.js` - Relevant file suggestions

**Templates** (`templates/`, bundled via package.json `files`):
- `base_prompt.md` - Initial base prompt template
- `instructions.md` - File request workflow context (installed by `p init`)
- `personas/*.md` - Persona templates: backend, frontend, linux, php, javascript

**Utilities** (`src/utils/`):
- `yamlParser.js` - Pure Node.js YAML parser (no dependencies)
- `input.js` - `InputCollector` class wrapping readline
- `gitignore.js` - Pattern matching for ignore files
- `tree.js` - Directory tree generation
- `slugify.js` - Filename slug generation

### Configuration

All constants in `src/config.js`:
- `PROMPT_DIR: '.create-prompt'` - Working directory for all commands
- `TASK_PLACEHOLDER` - Placeholder string replaced with the user's task
- `PERSONAS_DIR` / `INSTRUCTIONS_TEMPLATE_PATH` - Bundled template paths
- `RELEVANCE.*` - Context relevance filtering settings
- `TASK_TYPES.*` - Task type detection settings
- `MESSAGES.*` - User-facing strings
- `ALWAYS_IGNORE` - Default ignore patterns
- `VERSION` - Must be kept in sync with package.json `version`

### Working Directory Structure

```
.create-prompt/
├── context/                 ← Markdown context files with YAML frontmatter
│   ├── persona.md           ← Installed by `p persona <name>`
│   ├── project_info.md      ← Created by `p init` (kept if exists)
│   ├── project_structure.md ← Regenerated by `p ps`
│   ├── recent_activity.md   ← Regenerated by `p ps` (git branch + commits)
│   └── instructions.md      ← Created by `p init` (kept if exists)
├── base_prompt.md           ← Assembled from context files (see below)
├── ignore_files.txt         ← Structure tree ignore patterns
├── requested_files.txt      ← Input file list for `p fm`
└── results/                 ← Generated prompt files
```

### Data Flow for `p` (createPrompt)

```
User Input → collectUserInput()
                    ↓
           contextService.loadContext()
                    ↓
           templateService.buildFromBasePrompt()  ← fills title, date, task placeholder
                    ↓
           createPromptFile() → .create-prompt/results/
```

### Base Prompt Rebuild (`p ps`, `p persona`, `p init`)

`contextService.rebuildBasePrompt()` regenerates `base_prompt.md`:
header (title/date/tags placeholders) → context files sorted by `priority`
frontmatter (high → medium → low, alphabetical tiebreak) → preserved
`## History` → Task/Constraints/Expected Output scaffold.

## Conventions

- **No external dependencies** - Pure Node.js only
- **Async/await** throughout
- **camelCase** for files and functions
- All paths use `config.PROMPT_DIR` constant
- Commands export a single async function
- Services export multiple named functions
